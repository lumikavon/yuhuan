# 低于 95 分迭代协议

## 核心规则

裁判评分低于 95 分时，主持人不得输出最终结论。必须迭代。

## 迭代步骤

1. 读取裁判扣分项。
2. 将扣分项映射到顾问职责。
3. 生成 `Revision Brief`。
4. 定向召回 1-4 位顾问。
5. 顾问只补强对应缺口，不重写全部观点。
6. 主持人合并补强内容，重写候选报告。
7. 裁判重新评分。
8. 重复直到 `>=95`。

## Revision Brief 模板

```markdown
# Revision Brief vN

## 当前分数
...

## 未通过原因
...

## 需要召回的顾问
| 顾问 | 补强任务 |
|---|---|

## 禁止事项
- 不要重复上一轮观点。
- 不要为了高分捏造事实。
- 不要忽略硬性封顶规则。
```

## 无法达到 95 的处理

如果连续 3 轮仍无法达到 95，主持人必须判断原因：

1. **信息缺失**：向用户提出阻塞问题。
2. **风险不可消除**：输出“不建议行动”或“只建议低风险试验”。
3. **价值冲突无法代替用户选择**：清晰呈现冲突，让用户选择价值优先级。

不得伪造 95 分。
